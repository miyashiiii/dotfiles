# モデル / effort 切り替え基準

Claude Code のセッションで effort・モデルの変更を提案するときの基準。
`~/.claude/CLAUDE.md` の「うながし」ルールから参照される。提案時は必ず「いまの値」と「おすすめの変更先」を添える。

## 原則
- **effort が第一の調整ダイヤル**（思考量＝タスク難易度に合わせる）。
- **モデルが第二の大きいレバー**（地頭・知識・価格。Opus > Sonnet > Haiku）。
- 動かす順は「**まず effort → 足りなければモデル**」。

## effort
- `low`: 単純・定型・サブエージェント
- `high`: 知能を要する作業の基準ライン（品質とトークン効率のスイートスポット）
- `xhigh`: コーディング/エージェント用途（Claude Code 既定）
- `max`: 最難関で正確さ最優先のときだけ。**考えすぎ(overthinking)注意・非単調**（高いほど良いとは限らない）
- Opus 4.8 は **high を基準に上下**し、反射的に xhigh/max にしない。

## モデル
- `Haiku`: 機械的・定型（整形・単純置換・定型コミット・ドキュメント生成）
- `Sonnet`: 中庸。コスト重視の既定候補
- `Opus`: 高度な判断・知識・複数ファイル横断・設計

## 使い分けの型
- 簡単 → effort を下げる（ほぼノーリスク）→ Opus の地頭が要らなければモデルも下げる。
- 難しい → まず effort を上げ、足りなければ Opus に上げる（過設計防止に effort は上げ過ぎない）。

## 注意（現在値の読み方）
実セッションの effort/モデルは確実には読めない。`--effort` / `/config` / `/effort` / `/model` での変更はセッション限り、`~/.claude/settings.json` の `effortLevel` は新セッションの既定にすぎない。判然としなければ既定値ベースで、その旨を断っておすすめ先を示す。
